Ng TECH

Ng TECH is a basic portal for tech news powered by Hacker News. It is realized with Angular and designed with Angular Material. More details on the Github repo.

Project requirements

  • Use of Hacker News API.
  • Displaying of the latest news, with their general info.
  • Displaying only 10 news items at a time.
  • Using a button (Load more), view the info of the next 10 news items.

Look for a Book

Project requirements

  • Search function.

Angular Social

Angular Social is a basic simulation of a social network powered by Go REST. It is realized with Angular and designed with Angular Material. More details on the Github repo.

Project requirements

  • Login function (with Go Rest Bearer Token).
  • Viewing users with basic information. Ability to view a number of records of your choice and perform searches. Ability to add or remove a user.
  • User's detail page with all the available information, list of posts of the displayed user and the comments relating to each post. Ability to add a comment to each post.
  • View all posts in the system, with the possibility of carrying out searches. It must be possible to view comments on each post. It must also be possible to insert new posts.
  • Implement unit tests with at least 60% coverage.
  • Separate functionality into modules, downloaded in lazy loading.
  • Logout function.

Planty of Food APIs

This Node.js project recreates the JSON RESTful APIs for a hypothetical platform that manages purchasing groups for the Planty of Food company. Being a Node.js project, it was created in JavaScript with the help of the Express framework, the use of a MongoDB database for data storage and the Mongoose library to facilitate interaction with it. For unit tests, Mocha, Chai and Sinon were used instead. More details on the Github repo.

Project requirements

  • Respect the REST architecture, in particular the naming, methods and response status codes.
  • Insertion, modification and deletion of a product sold that has a characteristic: the name.
  • Insertion, modification and deletion of a user's details with the following characteristics: name, surname, email.
  • Insertion, modification and deletion of a sales order which will have the following characteristics: the products of that order, the users who are part of that order.
  • Viewing of all orders, filtering and viewing by date of entry of the order and by the products contained in it.
